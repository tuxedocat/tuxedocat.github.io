GNU screen: Run your commands even sudden connection lost occurs

January 12, 2012
nix

Assuming your system is Unix like systems, and screen is installed.

  1. Create sessions

    • $screen
    • (introduction will appear)
    • press enter

  2. Basic commands

    • C-a c: create new session
    • C-a w: show current sessions
    • C-a [0-Z]: move to other sessions
    • C-a d: detach screen
    • C-a k: kill screen

  3. Restoring sessions if connection lost occurs

    • $screen -ls: which shows list of screen sessions like this:

      There are screens on:
    pid.tty.host       (DATE1)   (Attached)
    pid.tty.host       (DATE2)   (Detached)
2 Sockets in /var/run/screen/S-$USERNAME.

    • $screen -r [pid.tty.host]: re-attach screen

    • $screen -D [pid.tty.host]: detach and logout screen
