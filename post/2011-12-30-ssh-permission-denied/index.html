Solving Permission Denied

December 30, 2011
nix

Problem

When logging in to server using public key authentication, I couldn’t log in.

ssh -vvv foo@bar
# omitted
debug1: Authentications that can continue: publickey
debug2: we did not send a packet, disable method
debug1: No more authentication methods to try.
Permission denied (publickey).

Popular solution

  • make sure permission of ~/.ssh is 700
  • also ~/.ssh/* is 600
  • this would be a pitfail: /home/xxx/ should be 755

Afterall…

The solution above didn’t work for me (Mac OS X 10.7.2, ssh-key type rsa2). It seems I’m just putting wrong key in .ssh
